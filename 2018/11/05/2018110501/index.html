Hexo 博客搭建，关键点整理

一、安装 Git

二、安装 Node.js

三、安装 Hexo

安装完 Node.js 后，再命令窗口执行 npm i -g hexo，之后可全局使用 Hexo 命令

四、初始化博客

新建一个文件夹，在该文件夹下执行 hexo init

目录结构

  • node_modules：是依赖包
  • ublic：存放的是生成的页面
  • scaffolds：命令生成文章等的模板
  • source：用命令创建的各种文章
  • themes：主题
  • _config.yml：整个博客的配置
  • db.json：source 解析所得到的
  • package.json：项目所需模块项目的配置信息

五、绑定 GitHub

  1. 登录个人 Github
  2. New repository (xxx.github.io)，xxx 必须与前面的名称相同，点击创建
  3. git 命令：git config --global user.name "xxx"
  4. git 命令：git config --global user.email "xxx"
  5. 创建 SSH：ssh-keygen -t rsa -C "youremail@example.com"
  6. 进入 C:\Users\xxx.ssh 目录，打开 id_rsa.pub 文件，将其中的文本复制到 Github 中（头像 - Setting - SSH and GPG keys - New SSH key）
  7. 验证：ssh -T git@github.com (出现 continue connecting 时输入 yes 即可，最后出现 success 表示成功)

六、修改配置

修改 _config.yml 配置文件中的配置

  1. 新建文章：hexo new “xxx”，编辑文章
  2. 在 source 目录下新建一个文件 CNAME，编辑内容 www.xxx.com

执行以下命令

访问 https://xxx.github.io 成功（需等待几分钟生效）

九、绑定个人域名（阿里云）

ping 一下自己的 Github 域名 xxx.github.io，并记录 IP 地址

  1. 登录阿里云控制台
  2. 域名 - 解析
  3. 添加一个（记录类型：A，主机记录：@，解析路线：默认，记录值：IP）记录
  4. 添加一个（记录类型：CNAME，主机记录：www，解析路线：默认，记录值：xxx.github.io）记录
  5. 访问个人域名 www.xxx.com
  6. 访问成功（需等待几分钟生效）
